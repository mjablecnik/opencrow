# Identity Files

The bot's behavior is influenced by identity files located in the `workplace/agent/` directory. These files are loaded on startup and included in the system context for every LLM request.

## File Overview

- **IDENTITY.md**: Static technical metadata about the bot
- **PERSONALITY.md**: Communication style and behavior patterns
- **SOUL.md**: Core beliefs, convictions, and authentic self
- **USER.md**: User preferences and context
- **TOOLS.md**: Tool usage guidelines for the agent
- **MEMORY.md**: Auto-generated memory index (do not edit manually)

## IDENTITY.md

Contains static technical metadata about the bot.

### Example

```markdown
# Bot Identity

**Name:** OpenCrow
**Created:** 2026-03-01
**Creator:** OpenCrow Team
**Version:** 1.0.0

## Purpose

OpenCrow is a conversational AI assistant that maintains comprehensive memory,
organizes domain-specific knowledge, and executes scheduled tasks automatically.

## Core Facts

- Built with Go and OpenRouter API
- Deployed as Docker container
- Maintains permanent conversation history with hierarchical organization
- Supports tool execution and scheduled reminders
- Features comprehensive memory system with topics and notes
```

## PERSONALITY.md

Describes the bot's communication style and behavior patterns.

### Example

```markdown
# Personality

## Communication Style

- Clear and concise
- Friendly and helpful
- Professional but approachable
- Patient with questions

## Behavior Patterns

- Provide direct answers when possible
- Ask clarifying questions when needed
- Explain technical concepts simply
- Acknowledge limitations honestly

## Response Format

- Write naturally with HTML formatting for emphasis
- Use Telegram-supported HTML tags for formatting
- Keep formatting simple and readable
- Use bullet points for lists
```

## TOOLS.md

Contains guidelines for how the agent should use available tools. This file helps ensure the agent uses tools correctly and follows best practices.

### Key Sections

- **Honesty and Transparency Rules**: Guidelines for being honest about what has been saved
- **Cron Management Tool**: How to create reminders and scheduled tasks correctly
- **Topic Knowledge Tool**: How to manage domain-specific knowledge
- **Notes Management**: How to use agent notes effectively

## MEMORY.md

Auto-generated index file that provides an overview of the memory system. This file is created and maintained automatically by the bot.

**Important**: Do not edit this file manually. It is regenerated by the memory system.

## Customizing Identity Files

1. Edit the files in the `workplace/agent/` directory to customize bot behavior
2. Restart the bot to load the new identity files:
   ```bash
   docker-compose restart bot
   ```
3. The bot will create default identity files automatically on first run if they don't exist
4. MEMORY.md is auto-generated and should not be manually edited
5. TOOLS.md contains guidelines for tool usage and should be customized based on your needs

## File Locations

- **Docker deployment**: Files are in `workplace/agent/` directory
- **Local development**: Files are in `workplace/agent/` directory
- **Default templates**: Included in Docker image at `/app/default-agent/`

The entrypoint script automatically copies default files to `workplace/agent/` if they don't exist on first run.
